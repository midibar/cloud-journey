# Yoga Studio – System Problems

******
1 Manual Reservation Management


The reservation process currently depends on manual work by the studio owner.

Students contact the studio through WhatsApp or email to request a class.

The studio owner then manually records the reservation in an Excel spreadsheet.

This creates operational overhead and slows down the booking process.

******
2 Risk of Human Error


Because most information is entered manually, the system is vulnerable to human error.

Possible mistakes include:
- incorrect reservation entries
- incorrect attendance recording
- incorrect pass tracking
- incorrect payment recording

******
3 Owner as Central Processor


Most operational processes depend on the studio owner.

The owner currently performs tasks such as:

- reading booking requests
- recording reservations
- updating attendance
- recording payments
- checking pass validity

This creates a bottleneck in the system.

******
4 Multiple Data Sources


Operational data is currently spread across multiple systems.

These include:

- Excel spreadsheets
- Airtable (partially implemented)
- Paper attendance sheets

This fragmentation increases the complexity of managing the studio's data.

******
5 Lack of Automation


The system lacks automation for core operations.

Examples include:

- no automatic reservation system
- no automatic pass validation
- no automatic class capacity control
- no automated attendance tracking
